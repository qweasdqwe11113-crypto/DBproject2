SeqScan 简要答辩
SeqScanOperator.java 做的是全表顺序扫描：

按页号从 1 到 totalPages 走
每页里按槽号从 0 到 recordsPerPage-1 走
用位图判断槽位是否有效（BitMap.isSet）
有效就按 RID(page, slot) 读出记录并返回 tuple

实现思路：

用户输入 SQL 后，先在 LogicalPlanner 里处理 `FROM table`，把它转成 `LogicalTableScanOperator`，表示逻辑上要从这张表读数据。

接着 PhysicalPlanner 会把这个逻辑扫描节点转成物理算子。当前项目里如果没有可用索引，就直接生成 `SeqScanOperator`，所以表扫描默认走顺序扫描。在 PhysicalPlanner.java 里，handleTableScan(...) 对普通表扫描直接 return new SeqScanOperator(tableName, dbManager);

真正执行时，DBEntry 会调用：

`Begin() -> hasNext() -> Next() -> Current()`

SeqScan 的核心做法就是按 `page -> slot` 顺序扫描整张表：

1. `Begin()`
打开表文件，读取总页数和每页记录数，并把游标初始化到第一页第一个槽位。

2. `hasNext()`
从当前页和当前槽开始往后找，用 bitmap 判断这个槽位是否真的有记录；如果这一页扫完了，就翻到下一页继续找。

3. `Next()`
找到下一条记录后，用 `RID(pageNum, slotNum)` 去读出这条 Record，并把游标往后移动。

4. `Current()`
把当前 Record 包装成 `Tuple` 返回给上层算子。
 physicalOperator.Begin();
                    while (physicalOperator.hasNext()) {
                        physicalOperator.Next();
                        Tuple tuple = physicalOperator.Current();
                        Logger.info(getRecordString(tuple));
                        Logger.info(getSperator(physicalOperator.outputSchema().size()));
                    }
                    physicalOperator.Close();
                    dbManager.getBufferPool().FlushAllPages("");

用到的主要文件
LogicalPlanner.java
LogicalAggregateOperator.java
PhysicalPlanner.java
AggregateOperator.java
SeqScanOperator.java
DBEntry.java
这些文件各自做什么
LogicalPlanner：识别 SELECT COUNT(...)，并创建 LogicalAggregateOperator。
isAggregateQuery(...) 判断是不是聚合查询（支持 COUNT/MAX/MIN）。
getAggregateFunctionName(...) 拿函数名（COUNT）。
getAggregateExpression(...) 拿参数表达式（如 * 或某列）。
LogicalAggregateOperator：逻辑层的“聚合节点”，只保存 child + functionName + aggregateExpr。
PhysicalPlanner：把逻辑聚合节点转换成物理执行节点 AggregateOperator。
AggregateOperator：真正执行 COUNT 的地方。
Begin() 里根据函数名分发到 computeCount()
computeCount() 循环读 child 的 tuple 并计数
Current() 返回单行结果 TempTuple([count])
SeqScanOperator：常见 child，负责把表行一条条吐给 AggregateOperator。
DBEntry：统一驱动执行 Begin -> while(hasNext){Next;Current} -> Close 并打印结果。
串起来（COUNT 执行链）
SQL 进入 LogicalPlanner.handleSelect。
发现是 COUNT(...)，在原来的 scan/filter/join 之上包一层 LogicalAggregateOperator。
PhysicalPlanner.generateOperator 把它变成 AggregateOperator(child, "COUNT", expr)。
执行时 AggregateOperator.Begin() 调 child.Begin()，然后在 computeCount() 中不断：
child.hasNext() -> child.Next() -> child.Current()
满足计数条件就 count++
扫描结束后，AggregateOperator 只产出 1 条结果（计数值）。
DBEntry 打印这一条结果。
COUNT 的语义处理（你代码里的细节）
COUNT(*)：aggregateExpr == null 或识别为 */AllValue，每行都计数。
COUNT(expr)：tuple.evaluateExpression(expr) != null 才计数（符合 SQL 里忽略 NULL 的思路）。
